## Custom MCP Server for Alicloud

Since Alicloud doesn't have an official MCP server yet, you can create a simple wrapper:

**Create:** `~/.claude/mcp-servers/alicloud-mcp.sh`

```bash
#!/bin/bash
# Alicloud MCP Wrapper

case "$1" in
  "list-instances")
    aliyun ecs DescribeInstances
    ;;
  "list-buckets")
    aliyun oss ls
    ;;
  *)
    echo "Usage: $0 {list-instances|list-buckets}"
    exit 1
    ;;
esac
```

**Install:**
```bash
chmod +x ~/.claude/mcp-servers/alicloud-mcp.sh

claude mcp add --scope project --transport stdio alicloud \
  -- ~/.claude/mcp-servers/alicloud-mcp.sh
```

## Troubleshooting

### MCP Server Not Working

1. **Check server status:**
   ```
   /mcp
   ```

2. **View logs:**
   ```bash
   tail -f ~/.claude/logs/mcp-*.log
   ```

3. **Reinstall server:**
   ```bash
   claude mcp remove <server-name>
   claude mcp add ...
   ```

### AWS MCP Issues

- **Error:** "Unable to locate credentials"
  - **Fix:** Run `aws configure` or set `AWS_PROFILE` env var

### Azure MCP Issues

- **Error:** "No subscription found"
  - **Fix:** Run `az login` and `az account set --subscription <id>`

### GCP MCP Issues

- **Error:** "Application Default Credentials not found"
  - **Fix:** Run `gcloud auth application-default login`

### PostgreSQL MCP Issues

- **Error:** "Connection refused"
  - **Fix:** Ensure DATABASE_URL is correct in .env
  - **Fix:** Check RDS security group allows your IP

### Kubernetes MCP Issues

- **Error:** "Unable to connect to cluster"
  - **Fix:** Run `aws eks update-kubeconfig --region ap-south-1 --name vulnerability-eks-cluster`

## Security Best Practices

1. ✅ **Never commit .env** to Git
2. ✅ **Use least-privilege IAM roles** for cloud access
3. ✅ **Rotate database passwords** regularly
4. ✅ **Use read-only access** when possible
5. ✅ **Enable MFA** for cloud provider accounts
6. ✅ **Audit MCP access logs** monthly
7. ✅ **Restrict MCP servers** to trusted packages only
