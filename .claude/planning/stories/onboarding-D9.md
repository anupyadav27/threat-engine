## Context

When the user selects `account_type = vulnerability` in the wizard (D8), the wizard diverges from the credential form flow into an agent install flow. Instead of entering credentials, the user is shown a generated install command with an embedded agent token (generated by C4's PKCE endpoint). The UI polls the agent's connection status — when the agent phones home, the status indicator flips from Pending to Connected. This story builds the agent install step component that plugs into the D8 wizard at the `vulnerability` account type branch. The PKCE token generation and heartbeat endpoint are live from C4.

## Technical Notes

**PKCE code challenge generation (client-side, before calling agent-token endpoint):**
```ts
// lib/pkce.ts
import { sha256 } from '@noble/hashes/sha256';  // or Web Crypto API

export function generatePKCE(): { verifier: string; challenge: string } {
  const verifier = crypto.randomUUID().replace(/-/g, '') + crypto.randomUUID().replace(/-/g, '');
  const hash = sha256(new TextEncoder().encode(verifier));
  const challenge = btoa(String.fromCharCode(...hash))
    .replace(/\+/g, '-').replace(/\//g, '_').replace(/=/g, '');
  return { verifier, challenge };
}
```

**Agent token request:**
```tsx
const { verifier, challenge } = generatePKCE();
const resp = await fetch(`/gateway/api/v1/cloud-accounts/${accountId}/agent-token`, {
  method: 'POST',
  headers: { 'X-PKCE-Verifier': verifier },
});
const { install_command } = await resp.json();
setInstallCommand(install_command);
// Do NOT persist verifier or raw token beyond this component's local state
```

**AgentStatusPoller:**
```tsx
// components/onboarding/AgentStatusPoller.tsx
const MAX_POLL_SECONDS = 300; // 5 minutes
let elapsed = 0;

const pollStatus = async () => {
  const resp = await fetch(`/gateway/api/v1/views/agent_status?account_id=${accountId}`);
  const data = await resp.json();
  setStatus(data.status);
  if (data.status === 'connected') {
    clearInterval(poller);
    setTimeout(() => onConnected(), 3000);
  }
  elapsed += 5;
  if (elapsed >= MAX_POLL_SECONDS) {
    clearInterval(poller);
    setTimedOut(true);
  }
};
useEffect(() => {
  const poller = setInterval(pollStatus, 5000);
  return () => clearInterval(poller);
}, []);
```

**BFF `view_agent_status` handler:**
```python
# onboarding_cloud_accounts.py
def view_agent_status(account_id: str, auth_context: dict) -> dict:
    headers = {"X-Auth-Context": json.dumps(auth_context)}
    resp = requests.get(
        f"{ONBOARDING_ENGINE_URL}/api/v1/cloud-accounts/{account_id}/agent-status",
        headers=headers,
        timeout=5,
    )
    if resp.status_code == 404:
        return {"status": "pending", "last_heartbeat": None}
    resp.raise_for_status()
    return resp.json()
```

**Onboarding engine `GET /api/v1/cloud-accounts/{id}/agent-status` endpoint** (add to cloud_accounts.py if not present):
```python
@router.get("/cloud-accounts/{account_id}/agent-status")
async def get_agent_status(
    account_id: UUID,
    auth: AuthContext = Depends(require_permission("cloud_accounts:read")),
    db=Depends(get_db),
):
    reg = await db.fetchrow(
        "SELECT status, last_heartbeat FROM agent_registrations WHERE account_id = $1 AND tenant_id = $2",
        str(account_id), auth.tenant_id
    )
    if not reg:
        return {"status": "pending", "last_heartbeat": None}
    return {"status": reg["status"], "last_heartbeat": reg["last_heartbeat"]}
```

**Token security:** The raw token is returned in `install_command` — store only in component state, never in wizard state or sessionStorage. Clear on component unmount.
